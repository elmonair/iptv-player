# PRODUCT.md

## Product

M2player is a logged-in IPTV streaming application. Users add their own playlist source, then browse and watch live TV, movies, and series.

This is a product surface, not a brand/marketing surface.

Design serves speed, clarity, and playback confidence.

## Register

product

## Category

Consumer media product
IPTV streaming interface
Cross-device playback app

## Users

Primary users are people who already have IPTV provider credentials and want fast access to content.

They are not exploring a new startup product. They are trying to:

- add a playlist quickly
- start watching quickly
- switch between live TV, movies, and series without friction
- resume content easily
- use the app on phone, desktop, and TV browsers

Many users are not technical. The interface should reduce hesitation and avoid jargon where possible.

## Usage Context

The app is used in these contexts:

- mobile phone, often one-handed
- desktop browser at home
- TV browser with keyboard or remote navigation

The interface must remain readable from a distance on TV and easy to scan on mobile.

## Core Jobs

Users come here to do a few concrete jobs:

1. Add a playlist
2. Open live TV, movies, or series
3. Continue watching something they started earlier
4. Switch provider playlist if needed
5. Reach playback with as few steps as possible

## Product Priorities

In order of importance:

1. Fast path to content
2. Clear navigation
3. Large readable controls
4. Playback trust and low confusion
5. Good behavior on mobile and TV

## Brand Voice

Clear
Calm
Practical

Optional tone qualities:

- direct, not chatty
- confident, not flashy
- helpful, not promotional

## Visual Direction

The UI should feel like a capable media tool, not a startup landing page.

Desired qualities:

- dark, cinematic base
- strong hierarchy
- large clear labels
- sparse but useful metadata
- high-contrast actions
- stable layouts that do not jump around

## Anti-References

Do not make this look like:

- a generic SaaS dashboard
- a marketing landing page
- a crypto app
- a glassmorphism concept UI
- a purple-gradient startup template
- a card-on-card-on-card AI layout

Avoid:

- long promotional copy
- repeated explanations
- small low-contrast labels
- decorative sections that delay content access
- dense settings-dashboard styling on content pages

## Navigation Principles

- Home page is a launch pad, not a billboard
- Add playlist flow should be short and calm
- Live TV, Movies, and Series should always feel like first-class destinations
- Continue Watching should appear early when useful
- Back behavior should feel obvious and predictable

## Home Page Guidance

The home page should emphasize:

1. Continue Watching
2. Direct entry to Live TV, Movies, Series
3. Playlist status and quick management
4. Minimal system/account information

The home page should not behave like a marketing hero section.

## Add Playlist Page Guidance

The add playlist page is a utility screen.

It should:

- be simple
- avoid repeated copy
- explain only what is necessary
- make the form feel trustworthy and easy

It should not read like onboarding for a new SaaS product.

## Interaction Guidance

- Mobile: thumb-friendly, large tap targets
- TV: focus-visible, directional-navigation friendly
- Desktop: fast scanning, low clutter

Hover is never enough on its own. Focus and active states must be obvious.

## Success Criteria

Design changes are successful when users can:

- understand where to go immediately
- add a playlist without confusion
- reach content quickly
- trust the app during playback
- use the interface comfortably on mobile and TV
